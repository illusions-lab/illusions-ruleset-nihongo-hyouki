# nh-l3-buntai-consistency — 文体（常体／敬体）の文書内統一

> ⚠️ ステータス: L3（LLM支援）・未実装。SDK が L3 を支持した後に実装予定。本ドキュメントは仕様の先行定義です。

- **ruleId**: `nh-l3-buntai-consistency`
- **ガイドライン**: 日本語表記（日本エディタースクール）`nihongo-hyouki`
- **レベル**: L3（LLM支援・文書全体分析）
- **想定 severity**: warning
- **適用モード案**: 小説 (`novel`) / 公用文 (`official`) / ブログ (`blog`) / 学術 (`academic`)

## 意図

口語体の文章には「……である、……だ」調（常体）と「……であります、……です」調（敬体）がある。
原則として混用することなくいずれかの文体で整理することが求められる。
この規則を文書全体で機械的に検証するには、すべての文末形式を把握したうえで「この文書が常体か敬体か」を判定し、逸脱した文を特定する必要がある。

## 書籍根拠

日本語表記（日本エディタースクール）第2章「文体について」§2 文体の整理（p. 8）

> 口語体にも、"……である、……だ"調の常体（普通体）と"……であります、……です"調の敬体（丁寧体）があるので、**混用することなくいずれかの文体で整理する**。

ポイント欄（p. 6）にも「原則として、混用することなくいずれかの文体で整理する」と明示されている。

## 動作原理

**LLM への入力**: 文書全体のテキスト（文末形式の一覧を含む）

**判断基準**:
1. 文書内の全文末を収集し、常体（〜だ／〜である／〜ない等）と敬体（〜です／〜ます／〜ません等）の比率を算出する。
2. 多数派の文体を「この文書の採用文体」と判定する。
3. 少数派の文体が混在する文を逸脱箇所として列挙する。
4. ただし、書籍が明示する許容例（箇条書き部分のみ常体、文の途中は常体 等）は除外する。

**出力**: 逸脱した文の位置・文末形式・推奨修正案

## なぜ L1/L2 で不可能か

- 常体か敬体かの判断は **文書全体の多数決** に依存する。単一の文だけを見ても逸脱かどうか分からない。
- 「〜です」の出現は「です」が敬体の証拠だが、全体が常体文書であれば指摘対象となり、逆なら正例となる。この「文脈依存の基準値」を正規表現や形態素解析の単純ルールで扱うことはできない。
- 許容例（箇条書き・文中途中）の判定にも文構造の理解が必要。

## 例

**positive（指摘すべき混在）**:

```
本研究は三段階で構成される。第一段階では調査を実施しました。第二段階では分析を行う。
```
→ 全体が常体（〜する）の中に「実施しました」（敬体）が混在 → 「実施しました」を「実施した」へ修正提案。

**negative（正常・単一文体）**:

```
本研究は三段階で構成されます。第一段階では調査を実施します。第二段階では分析を行います。
```
→ 全文が敬体で統一されているため指摘なし。

## 偽陽性・運用注意

- 許容混用パターン（箇条書き部分の常体化、敬体中の文中常体節）は除外ロジックに含める必要がある（書籍 p. 8 §2 敬体と常体を混ぜる例①〜③ 参照）。
- 小説（地の文＋会話文）では話者ごとに文体が異なることが多いため、会話文・引用文は評価対象から外す。
- 文書が短すぎる（5文未満）場合は判定精度が低いため適用を保留することを推奨。
