# nh-l3-suuji-policy-consistency — 数字表記方針の文書内統一（漢数字／アラビア数字）

> ⚠️ ステータス: L3（LLM支援）・未実装。SDK が L3 を支持した後に実装予定。本ドキュメントは仕様の先行定義です。

- **ruleId**: `nh-l3-suuji-policy-consistency`
- **ガイドライン**: 日本語表記（日本エディタースクール）`nihongo-hyouki`
- **レベル**: L3（LLM支援・文書全体分析）
- **想定 severity**: warning
- **適用モード案**: 小説 (`novel`) / 公用文 (`official`) / 学術 (`academic`)

## 意図

縦書きでは漢数字を使用する方針とアラビア数字を使用する方針があり、どちらかに決めて文書内で統一することが求められる。
この「文書が採用している数字表記方針」の判定と、その方針からの逸脱の検出は、文書全体の数字出現パターンを総合的に判断する必要があり、LLM による分析が適している。

## 書籍根拠

日本語表記（日本エディタースクール）第9章「数字の表記」§1 数字の表記、§2 縦書きの数字表記

数字にはアラビア数字・漢数字・ローマ数字など複数の方式があり、
書籍では文書ごとに一定の整理方針を立てて統一することが求められると説明されている。
縦書きでは漢数字方針またはアラビア数字方針のどちらかに決めて表記することとされ、
混在が見られる場合は見やすさ・読みやすさを基準に整理方針を定めることが推奨されている。

## 動作原理

**LLM への入力**: 文書全体のテキストおよび縦書き/横書きの設定情報

**判断基準**:
1. 文書内の数字表現（量を表す語）をすべて収集し、漢数字とアラビア数字の比率を算出する。
2. 多数派の表記方針を「この文書の採用方針」と判定する。
3. 少数派の数字表現を逸脱候補として列挙する。
4. 書籍が明示する除外カテゴリ（銀行口座番号・電話番号・西暦・統計数値・パーセント等）は評価対象から外す。
5. 熟語・固有名詞・慣用句内の数字（「一定」「二人三脚」等）も除外する。

**出力**: 方針違反の数字箇所・採用方針・推奨修正案

## なぜ L1/L2 で不可能か

- 「この文書が漢数字方針か、アラビア数字方針か」は文書全体を見て初めて判断できる。単文スコープでは方針自体が不明。
- 「三」が誤りかどうかは文書の採用方針次第（漢数字方針なら正しい、アラビア数字方針なら逸脱）。これは文脈依存の判断。
- 除外カテゴリ（電話番号・統計値等）の識別にも文脈理解が必要であり、正規表現だけでは電話番号と数量を区別できない場合がある。

## 例

**positive（方針違反）**:

```
（縦書き・漢数字方針の文書）
第三章では、調査対象を述べる。対象者は3名で、期間は二週間とした。
```
→ 「3名」がアラビア数字で他と不統一 → 「三名」への変更を提案。

**negative（方針統一）**:

```
（縦書き・漢数字方針の文書）
第三章では、調査対象を述べる。対象者は三名で、期間は二週間とした。
```
→ 漢数字で統一されているため指摘なし。

## 偽陽性・運用注意

- 書籍（第9章）が明示する除外カテゴリを必ず除く: 銀行口座番号、電話番号、振替番号、部屋番号、自動車・列車番号、西暦、緯度・経度、角度、標高、パーセント、統計数値、身長・体重。
- 混在が意図的な場合（見出しはアラビア数字、本文は漢数字 等）は文書構造を考慮して別々に評価する。
- 方針が明確でない短い文書（数字出現が5件未満）では判定を保留することを推奨。
- 2桁数字の「縦中横」等、組版上の制約による混在は指摘対象から外す。
